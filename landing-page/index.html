The D.E.R.E.K Workflow

Five mandatory phases. No shortcuts. Every task follows this proven process.

D

Design

Comprehensive analysis and planning before any code is written. Understand requirements, identify risks, design architecture, and create specifications.

Requirements Analysis Risk Assessment Architecture Design
E

Evaluate

Critical assessment of the proposed solution. Security implications, performance impact, maintainability concerns, and technical debt evaluation.

Security Review Performance Analysis Tech Debt Check
R

Review

Mandatory approval gate. Code review, security audit, compliance check. Nothing proceeds without explicit approval from reviewers.

Code Review Security Audit Approval Gate
E

Execute

Implementation phase with continuous monitoring. Write code, run tests, deploy changes. All actions logged and tracked for accountability.

Implementation Testing Deployment
K

Knowledge

Persistent memory system captures decisions, patterns, and context. Share project knowledge with team. Never lose important information.

Memory Retention Documentation Team Sharing
